What happens when you initialize a repository? Why do you need to do it?
  You add the files that will track the changed on the files. You need
  to make a repository to be able to use your git commands.

How is the staging area different from the working directory and the repository?
What value do you think it offers?
  It holds what files you want to commit and makes it easier to make commit
  without making to many at once.

How can you use the staging area to make sure you have one commit per logical
change?
  Only add things to the staging area that are part of one logical change.

What are some situations when branches would be helpful in keeping your history
organized? How would branches help?
  Branches can help add features and also be able to see previous changes to the
  files.

How do the diagrams help you visualize the branch structure?
  Help see which commit belong in the branch and what the parent commit is.

What is the result of merging two branches together? Why do we represent it in
the diagram the way we do?
  The result is a new branch that contains both set of changes. To show how
  the branches merge together.
